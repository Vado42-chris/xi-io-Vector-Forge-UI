## Core Principle

**"Surface the single most actionable item a user needs in any given situation."**

This means:
- **One clear action** at a time, not a list
- **Context-aware** based on workflow stage
- **Priority-based** (blockers > due today > reviews > in progress)
- **Real-time updates** as context changes

## Surface Priority Algorithm

### Priority Order (Highest to Lowest):

1. **Blockers** - Tasks you own that are blocked
2. **Due Today** - Tasks due within 24 hours
3. **Pending Review** - Tasks awaiting your review
4. **In Progress** - Tasks you're currently working on
5. **New Assignments** - Recently assigned tasks
6. **Suggestions** - AI-suggested actions

### Implementation:

```typescript
function getMostActionableItem(context: WorkflowContext): ActionableItem {
  // Priority 1: Blockers
  if (hasBlockedTasks(context.userId)) {
    return {
      type: 'resolve-blocker',
      task: getFirstBlockedTask(context.userId),
      urgency: 'critical',
      location: 'action-center',
    };
  }
  
  // Priority 2: Due Today
  if (hasTasksDueToday(context.userId)) {
    return {
      type: 'complete-due-task',
      task: getFirstDueTask(context.userId),
      urgency: 'high',
      location: 'action-center',
    };
  }
  
  // Priority 3: Pending Review
  if (hasPendingReviews(context.userId)) {
    return {
      type: 'review-task',
      task: getFirstReviewTask(context.userId),
      urgency: 'high',
      location: 'action-center',
    };
  }
  
  // ... continue priority order
}
```

## Context-Aware Surfacing

### Detection:

1. **Workflow Stage Detection:**
   - Analyze current UI state
   - Detect active tool/panel
   - Identify user intent

2. **User Role Detection:**
   - Check user permissions
   - Identify user role
   - Load role-specific features

3. **Task Context Detection:**
   - Check active task
   - Analyze task status
   - Identify blockers/dependencies

### Surface Rules:

1. **Primary Action:** Always in Action Center (top-right)
2. **Secondary Actions:** In relevant panels/components
3. **Supporting Items:** In tooltips, context menus
4. **Advanced Features:** In menus, on demand

## Validation Framework

### For Each Surface Element:

1. **Discoverability Test:**
   - Can user find it in < 5 seconds?
   - Is it in expected location?
   - Is it visually prominent?

2. **Accessibility Test:**
   - Keyboard accessible?
   - Screen reader compatible?
   - WCAG 2.1 AA compliant?

3. **Performance Test:**
   - Loads in < 100ms?
   - Interactions respond in < 50ms?
   - No lag or jank?

4. **Usability Test:**
   - Intuitive to use?
   - Clear feedback?
   - Error handling?

5. **Tracking Test:**
   - All interactions tracked?
   - Work sessions recorded?
   - Calculations logged?
